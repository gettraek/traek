Questions

---

**Q1:** What is BYOK?
```
BYOK stands for "bring your own key." You connect your own OpenAI or Anthropic
API key in Settings. træk sends your messages directly to that API — we never
see or store your conversations. You pay OpenAI or Anthropic directly for usage;
we charge only for the træk Playground app itself.
```

**Q2:** Is my data safe?
```
Your conversation data never touches our servers. Messages travel directly from
your browser to your chosen AI provider using your API key. Cloud sync (Pro and
Team) stores only the conversation graph structure — node positions, parent
relationships, and metadata — not the text content of your messages.
```

**Q3:** When does it launch?
```
We're targeting a public launch in Q2 2026. Waitlist members get early access
before the general public, and the first wave of invites goes to people who
refer the most friends. Join the waitlist to get notified the moment your
invite is ready.
```

**Q4:** How is this different from ChatGPT or Claude?
```
træk Playground isn't trying to replace ChatGPT or Claude — it's a different
interface that runs on top of them. Those tools are excellent linear chat
interfaces. træk is for when you need to explore multiple directions at once:
research, writing, planning, problem-solving. Think of it as a whiteboard for
AI conversations, not a replacement for chat.
```

**Q5:** Does it work with local models?
```
Not at launch. We're starting with OpenAI and Anthropic via BYOK. Local model
support (Ollama, LM Studio) is on our roadmap — if that's important to you,
mention it in the waitlist survey and it moves up the priority list.
```

## Copy Notes & Usage Guidelines

### Voice reminders
- **Always lowercase** `træk` — never `Træk` or `TRAEK`
- **No hedging** — "træk gives every message a location" not "træk can help manage"
- **Spatial vocabulary** — node, branch, canvas, path, navigate, map, pan, zoom
- **Second person, present tense** — "You branch. You zoom out." not "Users will be able to"
- **No filler** — cut any sentence that doesn't add information

### Subject line A/B options

Email 1 alternatives:
- "You're in — welcome to the waitlist"
- "Here's what træk does (and doesn't do)"

Email 3 alternatives:
- "Three things that make træk different"
- "Here's the canvas [screenshots inside]"

Email 5 alternatives:
- "Founding Members close this week"
- "Your invite is almost ready"

### Merge fields required
- `[First name]` — all emails
- `[referral link]` — all emails
- `[survey link]` — Email 4

### Deliverability notes
- Plain-text focused emails convert better with developer audiences
- Keep HTML version clean — no heavy images in Emails 1, 2, 4, 5
- Email 3 is the only one where screenshots add real value
- Use Resend's test mode to verify deliverability before launch
